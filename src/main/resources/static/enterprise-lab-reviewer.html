CI Dry-Run Evidence Artifact

Artifact name: container-dry-run-evidence-no-publish-no-sign

Evidence directory: target/container-dry-run-evidence/

What it proves: CI can capture local-only image identity, inspect/history/list output, Trivy summary, and dry-run summary evidence after the no-publish/no-sign container path runs.

What it does not prove: registry publication, container signing, provenance attestation, production deployment, or signed-container evidence.

What Is Proven

  • Maven tests pass in CI and local verification commands are documented.
  • Package build passes through Maven and CI smoke paths.
  • CI and CodeQL are required and passing on main according to the latest readiness and governance audit.
  • Packaged JAR smoke exists for local reviewer inspection.
  • Docker build/runtime smoke and Trivy run in CI without publishing or signing containers.
  • CI dry-run evidence artifact exists for no-publish/no-sign container review.
  • CODEOWNERS file exists for sensitive repo paths.
  • Documentation guardrails block overclaims in public readiness and reviewer docs.

What Is Not Proven

  • No production certification.
  • No enterprise-production readiness.
  • No registry publication.
  • No container signing.
  • No live AWS/cloud validation.
  • No private-network production validation.
  • No real tenant/IdP proof.
  • No production SLO/SLA proof.
  • GitHub governance settings are documented and recommended; repo files alone do not apply them.

Governance Posture

  • .github/CODEOWNERS covers workflows, CODEOWNERS, production Java code, tests, docs, evidence, Dockerfile, Maven build metadata, and README.
  • docs/MANUAL_GITHUB_GOVERNANCE_HARDENING.md documents repo-side ownership, recommended target settings, manual settings checklist, stale-review decision guidance, and evidence expectations.
  • The latest audit records an active main ruleset requiring CI and CodeQL.
  • Approval count and CODEOWNERS review enforcement remain manual GitHub settings decisions unless maintainers apply them later.
