# CI Artifact Consumer Guide

This guide shows reviewers how to consume LoadBalancerPro CI evidence without publishing a release. It covers GitHub Actions workflow artifacts for JaCoCo coverage, packaged artifact smoke output, CycloneDX SBOM files, and checksum comparison against a local build.

This path is release-free: no tags, no GitHub releases, no release assets, no `release-downloads/` changes, and no generated jars or checksums committed.

For operator command selection across CI artifacts, local verification, packaged jar startup, Maven exec, proxy profiles, and smoke helpers, see [`OPERATOR_INSTALL_RUN_MATRIX.md`](OPERATOR_INSTALL_RUN_MATRIX.md). For the hard stop before any future release process, see [`RELEASE_INTENT_CHECKLIST.md`](RELEASE_INTENT_CHECKLIST.md).

## Find The Latest Main CI Run

In GitHub, open Actions, choose the `CI` workflow, and select the latest successful `main` run. The expected build job is named:

```text
Build, Test, Package, Smoke
```

GitHub CLI:

```bash
gh run list --repo RicheyWorks/LoadBalancerPro --workflow CI --branch main --limit 5
gh run view <RUN_ID> --repo RicheyWorks/LoadBalancerPro
```

Use the run page or CLI output to confirm the CI workflow passed and the separate CodeQL workflow passed for the same mainline revision.

## Download Workflow Artifacts

Create a local review directory outside tracked source paths:

```bash
mkdir ci-artifacts
```

Download the coverage report:

```bash
gh run download <RUN_ID> --repo RicheyWorks/LoadBalancerPro --name jacoco-coverage-report --dir ci-artifacts/jacoco-coverage-report
```

Download the packaged jar smoke evidence:

```bash
gh run download <RUN_ID> --repo RicheyWorks/LoadBalancerPro --name packaged-artifact-smoke --dir ci-artifacts/packaged-artifact-smoke
```

Download the current CI SBOM artifact:

```bash
gh run download <RUN_ID> --repo RicheyWorks/LoadBalancerPro --name loadbalancerpro-sbom --dir ci-artifacts/loadbalancerpro-sbom
```

These are GitHub Actions workflow artifacts. They are not GitHub Release assets.

## Artifact Inventory

`jacoco-coverage-report`

- `index.html`: browser coverage report.
- `jacoco.xml`: machine-readable coverage report.
- `jacoco.csv`: coverage counters used by CI summary output.

`packaged-artifact-smoke`

- `artifact-smoke-summary.txt`: packaged jar smoke summary and required entry checks.
- `artifact-sha256.txt`: SHA-256 checksum for the jar built by that CI run.
- `jar-resource-list.txt`: sorted `jar tf` resource listing.

`loadbalancerpro-sbom`

- `bom.json`: CycloneDX JSON SBOM generated by current CI.
- `bom.xml`: CycloneDX XML SBOM generated by current CI.

If any expected workflow artifact is missing, inspect the workflow logs before treating the run as complete evidence.

## Inspect JaCoCo Evidence

Open the HTML report locally:

```bash
open ci-artifacts/jacoco-coverage-report/index.html
```

On Windows, open the file from Explorer or use:

```powershell
Start-Process .\ci-artifacts\jacoco-coverage-report\index.html
```

For exact counters, inspect:

```text
ci-artifacts/jacoco-coverage-report/jacoco.csv
ci-artifacts/jacoco-coverage-report/jacoco.xml
```

Only quote coverage percentages that come from the workflow logs or these downloaded JaCoCo files.

## Inspect Packaged Artifact Smoke Evidence

Read the smoke summary first:

```bash
cat ci-artifacts/packaged-artifact-smoke/artifact-smoke-summary.txt
```

Confirm the checksum output exists:

```bash
cat ci-artifacts/packaged-artifact-smoke/artifact-sha256.txt
```

Inspect the jar resource list for the required packaged resources:

```bash
grep 'BOOT-INF/classes/static/proxy-status.html' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
grep 'BOOT-INF/classes/static/load-balancing-cockpit.html' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
grep 'BOOT-INF/classes/application-proxy-demo-round-robin.properties' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
grep 'BOOT-INF/classes/application-proxy-demo-weighted-round-robin.properties' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
grep 'BOOT-INF/classes/application-proxy-demo-failover.properties' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
grep 'BOOT-INF/classes/com/richmond423/loadbalancerpro/demo/ProxyDemoFixtureLauncher.class' ci-artifacts/packaged-artifact-smoke/jar-resource-list.txt
```

Windows PowerShell equivalent:

```powershell
Select-String "BOOT-INF/classes/static/proxy-status.html" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
Select-String "BOOT-INF/classes/static/load-balancing-cockpit.html" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
Select-String "BOOT-INF/classes/application-proxy-demo-round-robin.properties" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
Select-String "BOOT-INF/classes/application-proxy-demo-weighted-round-robin.properties" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
Select-String "BOOT-INF/classes/application-proxy-demo-failover.properties" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
Select-String "BOOT-INF/classes/com/richmond423/loadbalancerpro/demo/ProxyDemoFixtureLauncher.class" .\ci-artifacts\packaged-artifact-smoke\jar-resource-list.txt
```

## Compare Local And CI SHA-256

Build a local jar if your workstation can resolve Maven dependencies:

```bash
mvn -B -DskipTests package
```

Generate a local checksum.

Windows PowerShell:

```powershell
Get-FileHash -Algorithm SHA256 .\target\LoadBalancerPro-2.5.0.jar
```

Unix:

```bash
sha256sum target/LoadBalancerPro-2.5.0.jar
shasum -a 256 target/LoadBalancerPro-2.5.0.jar
```

Compare the local digest to:

```text
ci-artifacts/packaged-artifact-smoke/artifact-sha256.txt
```

An exact match is useful integrity evidence when comparing the same artifact or a controlled rebuild with matching inputs. A local rebuild may differ if build inputs, timestamps, Maven plugin behavior, or environment details differ. Treat a mismatch as a prompt to compare source commit, build command, Java/Maven versions, and downloaded dependency set, not as automatic proof of a bad artifact.

## Release Candidate Checklist

- CI workflow passed on `main`.
- CodeQL workflow passed on `main`.
- CI reports zero skipped tests.
- `jacoco-coverage-report` artifact is present and inspectable.
- `packaged-artifact-smoke` artifact is present and inspectable.
- `artifact-smoke-summary.txt` reports required jar entries.
- `artifact-sha256.txt` exists for the CI-built jar.
- `jar-resource-list.txt` includes the proxy status page, cockpit page, proxy demo profiles, and `ProxyDemoFixtureLauncher.class`.
- `loadbalancerpro-sbom` is present when current CI is expected to upload SBOM files.
- Local SHA-256 is compared to CI SHA-256 when using the same artifact or a controlled rebuild.
- Operator docs and safety boundaries are reviewed.
- No tags, GitHub releases, release assets, `release-downloads/` changes, generated jars, or generated checksums are created for this checklist.

## Limits

- Workflow artifacts are CI review evidence, not official release assets.
- Checksums in `artifact-sha256.txt` describe the jar produced by that workflow run, not a signed release.
- Workflow artifacts can expire based on GitHub retention settings.
- Local rebuilds may not produce the same SHA-256 unless build inputs and environment are controlled.
- This guide does not add release publishing, cloud mutation, default proxy enablement, or production gateway guarantees.
